Scenario 1: Normal MFA

Expected Result: LOW Severity Alert

Simulates a legitimate user adding an MFA method during normal activity.

Steps:
  1. Sign in from a safe location (New York)
  2. Add a new MFA method
  3. System creates informational alert
Scenario 2: Risky MFA

Expected Result: CRITICAL Alert

Simulates an attacker adding MFA after impossible travel sign-in.

Steps:
  1. Sign in from Moscow (impossible travel)
  2. Wait 15 minutes (simulated)
  3. Add new MFA method
  4. System detects account takeover
Scenario 3: Takeover Pattern

Expected Result: CRITICAL Alert

Simulates attacker adding their MFA then removing victim's MFA.

Steps:
  1. Sign in from suspicious location
  2. Add attacker's MFA method
  3. Remove victim's original MFA
  4. System detects takeover pattern
